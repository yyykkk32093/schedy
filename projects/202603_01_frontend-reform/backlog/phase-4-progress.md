## 実装順序（推奨）

1. **UBL-17**（統計画面 + レポートAPI）— 分析機能の中核。集計クエリ → API → グラフUI
2. **UBL-19**（参加者推移）— UBL-17 の集計基盤を流用
3. **UBL-18**（欠席/キャンセル）— プッシュ通知基盤（UBL-14）完了後にアラート機能を追加
4. **UBL-20**（CSV出力）— 集計ロジックは UBL-17 と共通化可能
5. **UBL-21**（会計情報出力）— 決済データとの結合が必要
6. **UBL-22**（外部カレンダー）— 独立性が高く、任意のタイミングで実装可能

---

## 技術メモ

- **集計クエリ**: Prisma の `groupBy` / `aggregate` またはネイティブ SQL を活用
- **グラフライブラリ**: `recharts` or `chart.js`（React 対応）を検討
- **CSV 生成**: Node.js の `csv-writer` or `json2csv` を `backend/` に導入
- **PDF 生成**: `puppeteer` or `pdfkit` を検討（会計情報出力用）
- **iCal**: `ical-generator` ライブラリで `.ics` ファイル生成
- **Google Calendar API**: OAuth 2.0 認証 + Calendar Events API
