> 将来の自動課金導入やプッシュ通知基盤整備時に実装する。

| タスク                             | 元Phase       | 後回し理由                                                                                                                                                                  | 状態     |
| ---------------------------------- | ------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | -------- |
| off_session 課金失敗時プッシュ通知 | Phase 3 (3-4) | 手動支払いに変更。SetupIntent + off_session 自動課金を採用しなかったため不要。将来自動課金を導入する場合に `requires_action` / `card_declined` 時のプッシュ通知フローが必要 | ❌ 未着手 |
| 支払い催促通知（自動リマインド）   | Phase 3 (3-4) | 手動支払いフローでは MVP に不要。Schedule開始前に未払い参加者へ自動リマインドを送る機能。プッシュ通知基盤（FCM/APNs）整備後に実装                                           | ❌ 未着手 |
| プッシュ通知基盤（FCM / APNs）     | Phase 3 全般  | Phase 3 では Notification テーブルへの書き込み（簡易版）で対応。ネイティブプッシュ通知は後続 Phase で対応                                                                   | ❌ 未着手 |
