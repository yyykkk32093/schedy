## 1. 広告SDK選定

### 調査結果: クロスプラットフォーム統一は可能か

**完全に同一のライブラリでは不可能**。ただし **Google のエコシステム内で統一管理**は可能。

| プラットフォーム           | 使用SDK                           | 備考                                                                                    |
| -------------------------- | --------------------------------- | --------------------------------------------------------------------------------------- |
| Web（Vite SPA / ブラウザ） | **Google AdSense** (JSタグ)       | `<script>` タグ1つで導入。利用料無料                                                    |
| LIFF WebView               | **制限あり**                      | LINE WebView内はAdSenseポリシー上グレーゾーン。`liff.openWindow()` で外部ブラウザなら可 |
| iOS/Android ネイティブ     | **Google AdMob** (Mobile Ads SDK) | ネイティブ化時に追加                                                                    |
| ネイティブ内 WebView       | **WebView API for Ads**           | Mobile Ads SDK と WebView を連携させる公式API                                           |

**推奨戦略**: まず Web 向けに **AdSense** で実装 → ネイティブ化時に **AdMob** 追加 → **Google Ad Manager** でレポート・広告枠を一元管理

| プラン          | 広告     |

## 4. 収益モデル

### 広告ライブラリの利用料
**完全無料**。AdSense / AdMob は導入・利用にお金はかからない。

### お金が入る仕組み
```
広告主（企業）→ Google に広告費を支払う
    ↓
Google が広告ネットワーク経由でサイト/アプリに広告を配信
    ↓
表示・クリックに応じて Google → パブリッシャー（あなた）に報酬支払い
```

**Google の取り分**: 約 **32%**（AdSense の場合）。パブリッシャーの取り分は約 **68%**。

### 収益の種類と相場（日本市場）

| 指標    | 意味                     | 相場                                                           |
| ------- | ------------------------ | -------------------------------------------------------------- |
| **CPM** | 1,000回表示あたりの報酬  | バナー: **50〜300円** / インタースティシャル: **500〜2,000円** |
| **CPC** | 1クリックあたりの報酬    | **10〜50円**（一般） / **100〜300円**（金融・保険）            |
| **CTR** | 広告がクリックされる確率 | **0.5〜2%**                                                    |

### 試算例

FREEユーザー 1,000人、1人1日平均10画面表示:

| 項目             | 値                                        |
| ---------------- | ----------------------------------------- |
| 月間表示数       | 1,000人 × 10画面 × 30日 = **300,000表示** |
| バナー CPM 200円 | 300,000 / 1,000 × 200 = **60,000円**      |
| Google手数料 32% | -19,200円                                 |
| **手取り/月**    | **約40,800円**                            |

コミュニティ・スポーツ系は広告単価「中の下〜中」。ユーザーのエンゲージメント（滞在時間）が高いためフィード内広告の表示回数で稼ぐ形が効果的。

## 5. プライバシー・法令対応

### GDPR / Cookie Consent

- **GDPR** = EU一般データ保護規則。EU在住者の個人データ保護に関する法律
- **Cookie** = ブラウザに保存される小さなデータ。広告SDKはCookieでユーザーの閲覧行動を追跡し、パーソナライズ広告を配信する
- **Cookie Consent** = Cookieを使う前にユーザーの「同意」を得ること。サイト訪問時に「Cookieを使用します。同意しますか？」バナーを表示し、同意されるまで広告トラッキングを開始しない仕組み

**対応しないとどうなるか**:
- EU在住ユーザーがいる場合 → **最大2,000万ユーロ（約30億円）または年間売上の4%の罰金**
- **日本国内ユーザーのみなら当面リスクは低い**
- ただし日本の個人情報保護法も2022年改正以降Cookie規制を強化中。将来的には必要になる可能性あり
- **Google AdSense自体がConsent Mode対応を推奨**しており、未対応だとAdSense収益が減少する場合がある

### iOS ATT（App Tracking Transparency）

- Apple が **iOS 14.5** から導入したプライバシー機能
- アプリがユーザーを追跡（IDFA取得）する前に「"○○"が他社のAppやWebサイトを横断してあなたのアクティビティを追跡することを許可しますか？」ダイアログの表示を**義務化**

**対応しないとどうなるか**:
- **App Store 審査でリジェクト**（アプリ公開不可）
- ユーザーが「許可しない」を選ぶとパーソナライズ広告が出せない → **広告単価が30〜50%下落**
- **Web版（SPA）では関係なし**。iOS ネイティブアプリ化する時に必須

### 現実的な対応方針

| 時期           | 対応                                                                                     |
| -------------- | ---------------------------------------------------------------------------------------- |
| 今（Web SPA）  | 最低限の Cookie バナー + Google AdSense Consent Mode。日本向けなら法的緊急性は低いが推奨 |
| ネイティブ化時 | ATT ダイアログ実装は **必須**（審査通らない）                                            |

## 6. 技術的実装方針

| #   | 論点               | 方針                                                                     |
| --- | ------------------ | ------------------------------------------------------------------------ |
| 6-1 | 共通コンポーネント | `<AdBanner slot="xxx" />` を作り、各画面から呼ぶ。プラン判定は内部で行う |
| 6-2 | フィード内広告     | `<AdFeedItem />` を作り、リスト系画面で N 件ごとに挿入                   |
| 6-3 | 広告ブロッカー対策 | 検知しない（ユーザーの選択を尊重）                                       |
| 6-4 | パフォーマンス     | 広告の遅延ロード（Intersection Observer）でSPA初期ロードに影響させない   |
| 6-5 | エラーハンドリング | ロード失敗時は高さ0にcollapse。UIが崩れないことを保証                    |
