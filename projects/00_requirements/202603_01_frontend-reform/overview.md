# フロントエンド抜本的改革 — 全体プランニング

> 作成日: 2026-03-01
> Figmaで設計した17画面に基づき、現在の機能検証レベルUIをリッチなUIに段階的にリビルドする

---

## 📐 設計方針

### BottomNav 新構成（4タブ）

| タブ           | パス           | 備考                                |
| -------------- | -------------- | ----------------------------------- |
| ホーム         | `/home`        | 🆕 新規作成                          |
| コミュニティ   | `/communities` | 既存を刷新                          |
| アクティビティ | `/activities`  | カレンダー/タイムラインタブ付き新規 |
| チャット       | `/chats`       | チャット一覧を新規作成              |

### DM・スタンプ・通知の導線

| 機能                     | アクセス導線                                              |
| ------------------------ | --------------------------------------------------------- |
| DM                       | チャット（Chats）タブ内からアクセス                       |
| スタンプ（リアクション） | チャット内の各メッセージから（Slackライクなリアクション） |
| 通知                     | ヘッダーのベルアイコン 🔔                                  |

### 技術選定

| 項目         | 選定                                       | 理由                                                                                                                                      |
| ------------ | ------------------------------------------ | ----------------------------------------------------------------------------------------------------------------------------------------- |
| カレンダー   | **react-day-picker**（shadcn/ui Calendar） | shadcn/ui公式対応、Tailwindネイティブ、`modifiers`でアクティビティのある日にドット表示→日付タップで一覧表示が自然に実装可能、軽量（17kB） |
| アルバム機能 | フロント先行でUI構築                       | バックエンドAPIは後追いで追加                                                                                                             |

## 🔄 スレッド進行テンプレート

各スレッドでは以下の流れ：
1. **対象Mockup PNG共有** — Figmaで作成した画面イメージを添付
2. **差分確認** — 現在の実装との差分を特定
3. **設計** — ルーティング・UIコンポーネント・API連携を特定
4. **実装** — コード実装
5. **進捗md更新** — Phase別の進捗mdを更新

```
/home                              → Home画面（Phase 1）

/communities/create                → コミュニティ作成（Phase 2）
```
