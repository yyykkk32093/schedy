# Wave4 統合作業計画

> **最終更新**: 2026-04-12
> **ステータス**: ✅ Phase 0〜5+ 全完了

## 概要

wave4 は以下の **2系統** を統合して構成する。

| 系統                                          | 件数 | 内容                                                   |
| --------------------------------------------- | ---- | ------------------------------------------------------ |
| A. wave3レビュー指摘（wave3-carry-over）      | 24件 | wave3実装後の横断レビューで挙がったバグ修正・UI/UX改善 |
| B. wave2/wave3からの計画的繰越（旧wave4項目） | 9件  | 大規模新機能・リファクタ                               |

**方針**: A系統（レビュー指摘）を Phase 0〜3 で先行消化し、B系統（大規模機能）は Phase 4 以降で個別設計後に着手する。

**現状**: メタ情報がフラットに横並び。機能ボタンの位置が散在。

**目標レイアウト（上から順）**:
```
┌──────────────────────────────────┐
│ カバー画像                        │
├──────────────────────────────────┤
│ アバター + コミュニティ名         │
│ メンバー数 | 公開/非公開 | カテゴリ│  ← 名前の横/直下
├──────────────────────────────────┤

├──────────────────────────────────┤

├──────────────────────────────────┤
│ 📍 活動場所  🚉 最寄駅  📅 開催頻度│  ← 機能ボタンの下
│  (複数ある場合は重ねて展開UI)     │

│ 🗓 活動日  📊 レベル              │  ← 活動場所/駅の下

├──────────────────────────────────┤
```

## 横断的な設計判断ポイント

### D-1: 通知データモデルの拡張（Phase 3 で確定）

通知の情報拡充（C-20〜23）は、Phase 5+ の複数機能（W4-07 決済通知、W4-05 サブコミュニティ通知等）にも影響する。
Phase 3 で `metadata: Json?` 方式を採用する場合、Phase 5+ でも同じパターンを踏襲するため、**型定義の設計は拡張性を意識すること**。

```typescript
// 例: 通知メタデータの型定義（Phase 3 で確定）
type NotificationMetadata =
  | { type: 'WAITLIST_PROMOTED'; activityId: string; activityName: string; scheduleDate: string }
  | { type: 'SAME_DAY_CANCEL'; activityId: string; cancelledByName: string; scheduleDate: string }
  | { type: 'REFUND_REQUIRED'; activityId: string; targetUserName: string; amount: number }
  | ...
```

### D-2: CommunityLocation テーブル設計（Phase 3 で確定）

Phase 3（C-12）で新設する `CommunityLocation` テーブルは、Phase 5+（W4-01 ウィザード、W4-03 詳細検索）でも活用される。
設計時に以下を考慮:
- W4-03 の検索でインデックスが効くカラム構成にする
- W4-01 のウィザード詳細設定で追加されるエリア情報との整合性

### D-3: 退会理由の選択肢（Phase 3 で要確認）

C-25 の退会理由選択肢が wave3-carry-over.md に記載されていない（「選択肢は以下の通り」で途切れている）。
**⚠️ ステークホルダーへの確認が必要**。

### D-4: C-13 レイアウト改修と W4-01 ウィザードの関係

Phase 2（C-13）でコミュニティ詳細のレイアウトを改修するが、Phase 5+（W4-01）で詳細設定項目（年齢層・性別・レベル等）が追加される。
Phase 2 のレイアウトは **追加項目が入る余地を残した設計** にすること。

### D-5: アルバムの写真アップロードフロー（Phase 2 で確認）

C-15 で写真追加UIを実装するが、S3アップロードの既存フローを確認する必要がある。
