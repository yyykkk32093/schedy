## ✅ 確定した設計判断

| #   | 論点                                      | 決定                                                 | 理由                                                                                                                        |
| --- | ----------------------------------------- | ---------------------------------------------------- | --------------------------------------------------------------------------------------------------------------------------- |
| D-1 | ブックマークのDBマイグレーション          | Phase 3 冒頭で `announcement_bookmarks` テーブル新設 | Phase 3 の最初のタスク（3-1）で必要                                                                                         |
| D-2 | キャンセル待ち＋自動繰り上げの Phase 配置 | **Phase 3**（旧 Phase 4 から移動）                   | バックエンド実装済み（UseCase, API完備）。フロントUI接続のみで済む。参加ボタン系[11][12][13]をまとめて対応                  |
| D-3 | 支払い済みキャンセルのフロー              | **確認ダイアログ型（Bパターン）**                    | キャンセルは許可するが「返金は幹事に直接ご確認ください」の警告表示。管理者にOutbox通知。Stripe返金自動化は将来Phase 5で対応 |
| D-4 | チャット既読数の実装方式                  | **ハイブリッド型（Cパターン）**                      | 既読送信=WebSocket（`message:read` C→S）、既読数取得=REST（メッセージAPIに`readCount`付与）。既存Socket.io基盤を活用        |
| D-5 | マイク入力機能                            | **IdeaBox送り**                                      | 実装工数大。review.mdでもIdeaBox行き許容の記載あり                                                                          |
| D-6 | 言語選択（i18n）                          | **別案件（202603_06_i18n）に切り出し**               | ライブラリ未導入＋全テキスト抽出で工数大。本案件では日本語化のみ対応                                                        |
