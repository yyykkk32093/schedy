## C-07 いいねボタンが反応しない

- **分類**: バグ修正
- **優先度**: P1
- **変更対象**: フロント
- **変更レイヤー**: UI
- **由来**: wave3レビュー [7]
- **依存**: なし

### 現象
- お知らせ詳細画面でいいねボタンを押しても反応しない（UIが更新されない）

### 想定原因
- mutation のキャッシュ更新ロジック（React Query の `onSuccess` / `onMutate`）にバグがある可能性
- または mutation 自体が正しく呼び出されていない可能性

### 調査・修正方針
1. お知らせ詳細画面のいいねボタンの `onClick` → mutation 呼び出しを追跡
2. mutation の実行状態（`isPending`, `isError`, `isSuccess`）を確認
3. APIリクエストが送信されているか DevTools で確認
4. キャッシュ更新（`queryClient.setQueryData` / `invalidateQueries`）が正しく動作しているか確認

### 受入条件
- お知らせ詳細画面でいいねボタンを押すとUIが即座に更新されること
- いいね状態がAPIに正しく保存されること
- いいねの取り消しも正常に動作すること

### 関連ファイル（推定）
- `frontend/src/features/announcement/` — お知らせ詳細コンポーネント・いいね関連hooks

## C-05 検索フィルターが効いていない

- **分類**: バグ修正
- **優先度**: P1
- **変更対象**: フロント(+BE?)
- **変更レイヤー**: UI（+ 場合によりAPI）
- **由来**: wave3レビュー [5]
- **依存**: なし

### 現象
- お知らせタブで検索フィルターが効いていない（キーワード検索で絞り込みが動作しない）

### 設計判断
- **原因調査が先** — まずFE側のkeywordフィルターが2文字制限で空キーワードの場合の挙動を確認する
- 考えられる原因:
  - a) FEのフィルター条件ミス
  - b) BEのAPIクエリバグ
  - c) キャッシュの問題

### 調査・修正方針
1. FE側のフィルターロジック（keyword state → API query params）の流れを確認
2. 空キーワード・1文字・2文字での挙動差を確認
3. BE側APIのクエリパラメータ処理を確認（WHERE句にkeywordが正しく適用されているか）
4. 原因特定後に修正

### 受入条件
- お知らせタブでキーワード検索が正しく動作すること
- 空キーワード時は全件表示されること
- 検索結果がリアルタイムに反映されること

### 関連ファイル（推定）
- `frontend/src/features/announcement/` — お知らせ一覧・検索コンポーネント
- `backend/src/api/front/announcement/` — お知らせ検索API（BE側が原因の場合）
- `backend/src/application/announcement/usecase/` — 検索UseCase（BE側が原因の場合）

## C-01 既読数から自分自身を除外

- **分類**: バグ修正
- **優先度**: P2
- **変更対象**: 両方
- **変更レイヤー**: UI / API
- **由来**: wave3レビュー [1]
- **依存**: なし

### 現象
- お知らせ一覧の既読数に自分自身のカウントが含まれており、実質の「他メンバーの既読数」より1多い

### 設計判断
- **a) BE側でクエリから除外して返す** — レスポンスの `readCount` を正確にする
  - 既存APIの `readCount` を算出するクエリから、リクエストユーザー自身のレコードを除外
  - FE側の変更は不要（or 最小限）

### 仕様
- お知らせ一覧/詳細APIの `readCount` レスポンスから、リクエストユーザー自身の既読を除外
- 「自分を含まない」既読数を返す

### 受入条件
- お知らせ一覧の既読数に自分自身が含まれないこと
- 自分が既読にした直後、既読数が +1 されないこと（他メンバーが読んだ場合のみ +1）
- 既存APIの他の利用箇所に影響がないこと

### 関連ファイル（推定）
- `backend/src/application/announcement/usecase/` — 既読数取得ロジック
- `backend/src/api/front/announcement/` — APIレスポンス構築
- `frontend/src/features/announcement/` — 既読数表示コンポーネント

## C-17b アクティビティ更新APIの権限チェック（BE側）

- **分類**: 改善
- **優先度**: P2
- **変更対象**: バックエンド
- **変更レイヤー**: API / UseCase
- **由来**: wave3レビュー [17]（Phase 0 C-17 のBE側対応）
- **依存**: C-17（Phase 0 のフロントガード）

### 仕様
- Phase 0 のフロントガード（C-17）に加え、BE側でも権限チェックを実施
- アクティビティ更新/削除APIでリクエストユーザーの権限がADMIN未満の場合は 403 Forbidden を返却

### 受入条件
- MEMBER ロールでアクティビティ更新APIを直接呼び出した場合、403エラーが返ること

- エラーレスポンスに適切なメッセージが含まれていること

### 関連ファイル（推定）
- `backend/src/api/front/activity/` — アクティビティ更新/削除APIエンドポイント
