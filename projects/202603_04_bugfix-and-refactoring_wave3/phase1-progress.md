## W3-03 フリーテキスト最大文字数の棚卸し・統一

- **分類**: 改善
- **優先度**: P2
- **変更対象**: 両方
- **変更レイヤー**: UI / API（DTOバリデーション）

全フリーテキスト入力フィールドの maxLength を棚卸しし、BE（DTOバリデーション）とフロント（input maxLength / textarea maxLength）を統一する。

| Message（チャット） | content | 1000 | |
| User | displayName | 30 | |
| User | biography | 200 | |
| Expense | note | 200 | |
| VisitorProfile | memo | 200 | 将来追加予定 |

## W3-04 繰り返し/コピー/移動時の支払いステータス整理

- **分類**: 改善
- **優先度**: P2
- **変更対象**: 両方
- **変更レイヤー**: UseCase / Domain
- **由来**: 積み残し #11
- **依存**: なし

### 仕様
アクティビティの繰り返し生成・コピー・移動時に、Paymentレコードのステータスをどう扱うか統一ルールを定める。

| 操作 | Paymentの扱い | 理由 |
|------|--------------|------|
| **繰り返し生成** | Payment作成しない（参加者もコピーしないため） | 新規スケジュールに参加者はまだいない |
| **コピー** | Payment作成しない（参加者もコピーしないため） | コピー元とは独立した新規スケジュール |
| **移動（日時変更）** | 既存Paymentを維持 | 同一スケジュールの日時変更なので支払い状況は継続 |

### 受入条件
- 繰り返し生成で新規作成されたスケジュールにPaymentが紐づかないこと
- コピーで作成されたスケジュールにPaymentが紐づかないこと
- 日時移動後のスケジュールで既存のPayment情報が維持されていること
- 各操作後にアクティビティ詳細画面の支払い情報が正しく表示されること

### 関連ファイル（推定）
- `backend/src/application/schedule/usecase/` — コピー・移動UseCase
- `backend/src/application/activity/usecase/` — 繰り返し生成UseCase

## W3-06 アクティビティ削除時の通知設定

- **分類**: 改善
- **優先度**: P2
- **変更対象**: 両方
- **変更レイヤー**: UI / API / UseCase
- **由来**: 積み残し #5
- **依存**: なし

### 仕様
アクティビティ削除の確認ダイアログに通知方法の選択肢を追加する。

| 選択肢 | 動作 |
|--------|------|
| **お知らせとして投稿する** | Announcement を作成（「○○が削除されました」）+ プッシュ通知 |
| **プッシュ通知のみ** | Notification レコード作成 + プッシュ送信。Announcement は作成しない |
| **通知なし** | 何もしない。サイレント削除 |

- デフォルト選択: 「プッシュ通知のみ」
- 通知対象: 該当アクティビティの参加者全員（Participation.userId）

### 受入条件
- 削除確認ダイアログに3つのラジオ選択肢が表示されること
- 「お知らせとして投稿」選択時、削除後にお知らせ一覧に投稿が表示されること
- 「プッシュ通知のみ」選択時、通知は届くがお知らせ一覧には投稿されないこと
- 「通知なし」選択時、削除のみ実行されること

### 関連ファイル（推定）
- `frontend/src/features/activity/pages/ActivityDetailPage.tsx` — 削除ダイアログ
- `backend/src/application/activity/usecase/` — 削除UseCase
- `backend/src/application/announcement/usecase/` — お知らせ作成UseCase
- `backend/src/job/` — プッシュ通知送信
