## 各項目の詳細

### W4-01 コミュニティ作成: ウィザード化

**現状**: 1ページのフォームで基本情報のみ入力

**目標**: 3ページのウィザード形式 + 詳細設定

| ページ | 内容 |
|--------|------|
| 1. 基本情報 | 名前・説明・カバー画像 |
| 2. 参加設定 | 公開/非公開、検索対象設定、参加方式（自由参加/承認制/招待制） |
| 3. カテゴリ・タグ | カテゴリ選択、タグ入力 → 完了 or 詳細設定へ |
| 詳細設定（任意） | 最大メンバー数、年齢層（範囲）、対象性別（LGBTQ+含む）、活動エリア・最寄駅、活動頻度、活動曜日、推奨レベル（0〜8範囲） |

**DB変更（Community テーブル）**:
- `targetAgeMin` / `targetAgeMax` (Int?)
- `targetGender` enum拡張（LGBTQ+追加）
- `recommendedLevelMin` / `recommendedLevelMax` (Int? 0-8)
- `isSearchable` (Boolean)
- `activityDayOfWeek` は既存 CommunityActivityDay テーブル活用

**依存**: なし
**詳細設計**: wave4着手時に策定

### W4-07 キャンセル待ち繰り上げ: PayPay/クレジット決済連携

**現状**: wave3（W3-10）で現金デフォルト自動作成を実装予定

**目標**: 繰り上げ時に支払い方法を選択可能にする

| 支払い方法 | 動作 |
|-----------|------|
| 現金（デフォルト） | W3-10で実装済み。PENDING状態で自動作成 |
| PayPay | PayPay決済リンクを生成し通知で送信 |
| クレジットカード | Stripe Connect経由で決済。Stripe Connectオンボーディング必須 |

**Stripe Connect Express フロー**:
1. コミュニティ設定画面で「クレジットカード決済を有効にする」ボタン押下
2. Stripe ホスト型オンボーディングページにリダイレクト（Account Links API）
3. コミュニティ管理者が名前・銀行口座等を入力（Stripe が KYC を処理）
4. 完了後にアプリに戻る → `Community.stripeAccountId` が自動設定

**依存**: W4-02（Stripe基盤の整備）
**詳細設計**: W4-02完了後に策定

### W4-08 アクティビティ詳細パス: ネスト型リファクタ

**現状**: `/activities/:activityId?schedule=xxx`（トップレベル独立パス）

**目標**: `/communities/:communityId/activities/:activityId?schedule=xxx`（ネスト型）

**変更理由**:
- アクティビティはコミュニティ従属リソース（Activity.communityId は必須FK）
- 一覧(`/communities/:id/activities`)・作成(`/communities/:id/activities/new`) は既にネスト型
- 詳細だけトップレベルなのは一貫性に欠ける
- URL階層がドメインモデルの所有関係を正確に反映する

**変更箇所（推定8箇所+）**:
- フロントルーティング定義
- ナビゲーション箇所（FeedCard, ActivityCard, SearchResult, NotificationPage 等）
- 通知・ディープリンクのURL生成（バックエンド）
- LIFF / ネイティブアプリのディープリンクハンドラ

**依存**: LIFF / ネイティブアプリとの連携確認が必要
**詳細設計**: wave4着手時に影響範囲を洗い出し
